Changing Flask's default templates directory

Posted by : Kevin   Posted at : Wednesday, May 28 2014
Assume that you had to move all your Flask application's files and directories to another sub-directory named, let's say app/. In that case, now when you call the 'render_template' method for your route from the gateway file (which is outside your directory), Jinja2 will look for a folder named 'templates' in the current directory.

That being the case, I managed to change the default 'templates' folder like this :


import jinja2

# ..
app = Flask(__name__)
app.jinja_loader = jinja2.FileSystemLoader('app/templates')

# ..

Now, restart your application and it's done.
